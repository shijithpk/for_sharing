Hi Jacqueline,

I suppose the first part of the answer, is a question, and that is what are you trying to accomplish?
If you want to write code to run in people's web browsers, JavaScript is the language you need.
If you want to write scripts to accomplish a variety of tasks such as web scraping, manipulating text, and so on, Python is great for those things.
If you just want a language that can help you master basic programming concepts, I'd definitely say Python. It has a cleaner syntax, and basic structures such as loops, if statements, and functions are probably easier to get your head around in Python. Python's requirement to indent code blocks imposes a certain syntactic discipline and makes the code easier to read.
There are a lot more complicated answers than this, and when you get into more advanced programming, each has libraries that can make your work easier and more efficient.
Hope that helps

I learned Javascript first but it was out of desperation because there
was no other way to do interactivity at the time (most people were
using CGI and I had no access to it) -- knowing Javascript certainly
keeps you close to web page events and handling things in terms of
clicks and responses, but on the other hand, it's getting more and
more packaged into libraries that do different things in different
browsers -- the programming is not as pure as it was. But you learn
how code works, how it has to be referenced, how to debug...(why isn't
this working??? What can I do to find out???) all valuable.
 -- scripting like Python is handier for database work where you need
to need to run a series of queries based on other queries -- I guess
an example might be, "What are all the zip codes in this data and
what's the median value of x for each?" or, more complicated, "out of
all these time card entries, what are some of the longest time periods
of uninterrupted work?" that are hard to do with individual queries
but can be scripted. And nowadays R can do a lot of that. But Python
gets you off and running on programming where R does not necessarily
do that. --Tim Henderson

Javascript. But don't start learning in the browser. Start learning with node. You can focus on javascript without having to know anything about HTML, CSS, etc.

Getting set up with javascript is a lot easier than with Python these days. You can just 'npm init' and then start installing packages to let you do web scraping, text parsing, etc. etc.

And later, when you're ready to move to the web, all your skills will be directly transferable.

Don't get me wrong. Python's great. But javascript is more useful right now (and Python's 2.x/3.x issues and virtualenv nonsense are super messy).

P.S. re/callback hell, now that promises have been adopted into the language (and async functions are on the way), callback hell is easily avoidable.

I'll stick up for javascript here. For a beginner, I'd have no problem
recommending it over python.

As Daniel said, it's the lingua franca of the Internet.

For me it's way more forgiving than python. Forgetting to indent
something isn't likely to trip you up. You don't have to worry if
you're using version 2.6, 2.7 or 3. You don't have to be comfortable
with a command line or terminal interface, you can use it right in the
web browser you're comfortable with.

It's also very easy to see the results of your labour. When you're
learning and testing things out it's great to able to modify DOM
objects in your browser's console and see an instant result on the
screen. Or you can avoid HTML and CSS all together and just work in
node.

When you start to get more comfortable with it, you can use it to set
up and run your own server. You can use it to easily scrape the web.
You can use it for data visualization. Just look at all these examples
https://github.com/d3/d3/wiki/Gallery - all done with a javascript
library. Really, being able to do both client and server in the same
language is amazing, but probably not something someone just starting
out cares about.

Ultimately, I think if you look at who's investing in javascript now:
Apple, Google, Facebook, Microsoft, Mozilla, you'll find that
javascript is going to continue to grow in importance. Why not start
from there?

As a JS coder, I'd probably recommend Python for an absolute beginner.

If you learn via Node, it's going to require a lot of mental work to wrap your head around continuation-based code. I don't think that's great for beginners. If you learn in the browser, it's a cool space to figure things out, but it's harder to debug and you can't write simple file processing scripts, which are a good way to learn. D3 is not a good beginner experience--they'll mostly learn to copy and paste.

If someone learns the basics with Python, they won't have too many problems transferring over to JS. It's good for beginners to learn to indent properly, and Python will force them to do that. And there's a lot of good textbooks for learning Python as a beginner, while I've never been entirely happy with the JS textbook landscape.

I would say Python, especially if you're teaching a language so they can understand and build APIs. Python objects always made data seem more understandable to me, and being able to quickly write some loops and parse data in the terminal also made me super happy. Like, oh wow, I just made XYZ happen! THIS IS SO NIFTY! And with Flask and other frameworks (Django Rest Framework/Tastypie/etc.), you can also quickly make a REST API, as long as those you are teaching get the other programmatic fundamentals down.

Whereas, for me, doing things in the browser feels less permanent. Not to mention, as someone who has been programming in Python for seven years and who just had to build something in Node.js for the first time, Python was a lot easier to just dive into and build some shit vs command-line JavaScript.

ALL THAT BEING SAID: I teach a college course on storytelling through web development. I barely touch JavaScript, let alone Python, and just focus on the fundamentals of storytelling primarily through HTML/CSS/design in static pages instead of some super fancy app that hits 19 databases and makes 72 charts. Personally, APIs and frameworks and more functional programming languages are something I would save for much, much, much later, after folks have gotten some "make shit" self-esteem. Which is why I don't teach them at all in my class :)

Learn Python The Hard Way is still the best book on learning a programming language (other than just building shit) in my opinion, and going through that made learning other languages relatively a breeze, as it covers the basic grammar of all programming languages.

Good luck!

I’ve seen a few people here  say that JavaScript has to be run in a web browser. That’s no longer the case. Node is a JavaScript interpreter that lets you run JavaScript on the server (or on your local computer) just like Python or Ruby. And both Node and Python have libraries that make if fairly easy to build a data API.

That said, I think Python is probably the more accessible language for a beginner. You won’t have to deal with JavaScript idiosyncrasies like asynchronous programming that might be more difficult for a beginner to get their head around. Python would also be the choice if, in addition to building APIs, your main goal is data munging and analysis. Python has more developed libraries for that.

On the other hand, JavaScript would be the more marketable, if that’s a factor in your decision. Web development today is all JavaScript all the time.

Late to this, but I would say the answer is whatever your patient, supportive friends and colleagues use.  

I've found having people who can answer questions and validate your frustrations is the most important thing when learning something.  I no longer use any of the first programming languages that I learned, and that's ok.  Learning any language, and feeling confident and knowing how to ask the right questions made it relatively painless to learn new languages.

Python and JavaScript are both, at this point, capable general purpose programming languages, and you can build APIs with both.  I tend to agree with those that say Flask (Python) is easier to understand than something like Express (JavaScript), but I don't think the differences are so great that it trumps having people in your life who can answer questions or just tell you "no sweat, this is hard when I learned it too!"
