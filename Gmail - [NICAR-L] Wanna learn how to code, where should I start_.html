That depends on your background and your ambitions. If your ambition is to build digital "news products", learn JavaScript and at least one front-end development framework. I'm not a front-end developer but the two names I hear bandied about most are React and Angular (2), although Mozilla recommends Ember.

If your ambition is to work on the back end / light-weight data science, go with Python. The best way to do that is to install "Anacoda" and play around with IPython / Jupyter notebooks in the browser.

If your ambition is to learn deep-dive statistics, go with R. The newer versions of RStudio and associated libraries will do some amazing things - eBook authoring, dashboards, interactive apps, etc. - but they're essentially desktop bound. You *can* deploy them in the cloud but there's a dependency on having an R server running, which is a non-trivial (aka expensive) proposition compared to other back end options.

For your experience and work with data so far, either Python or R will be ideal as a first language, and other languages will come much easier after you've understood the core ideas behind programming. Python and R are more ideal than other languages because of their immense popularity and communities that are highly interested in data exploration, which means that their users have created and refined tools that make such work easier.

But between those two, no other factor is more important than your personal situation, which includes what classes/workshops/meetups are geographically/financially available to you, and more importantly, whether you have colleagues and friends who are either learning or are actively using a language on a daily basis. Learning programming is not more complicated than it was 20 years ago but there's a lot more variety in setups and assumptions such that, even with Google, it's difficult on your own to know the difference between what's hard (but worthwhile) and what's just some inconsequential detail that could nonetheless hold you back for hours or days...and it just takes a few of those to wreck your enthusiasm.

Even if either of these two languages were objectively 5 times better than the other, it still wouldn't make up for the value of having peer support/pressure to get you through the first phase of learning programming on your own. I wouldn't have high hopes for learning programming as a way to reach a higher level...it is extremely likely you'll end up accomplishing much less through programming, materially, than if you did things the old fashioned way, for the first couple of months at least. But it's not so much as gaining proficiency in a technical skill as it is understanding programming as a new way of thinking. Good friends/colleagues/social circles have much more impact in facilitating that than any quality inherent to a language.

​If you're new to programming in general, it really does help to learn basic programming concepts first and Python is a great way to do that. It's a much more natural language (more like english) than others and it's a great tool, even if you end up deciding JS or Java or R is what you want to dive into more deeply. 


Love Zeb Shaw's Python the Hard Way, and as a practice, he's got the idea.. try everything out, typing and repetition build muscle memory, learn by doing.. but I think the "active learning/project-based learning" approach of the Head First series is better. The basic intro to programming book in that series is in Python and the second book (Head First Python) in particular is great and looks at more complex data and approaches.


The problem with Code Academy and such is that it can create familiarity, but you eventually need to have a project or problem to solve that helps you really apply what you've learned. The Head First books do that better than some, which is why I like them, but to echo what other are saying.. give yourself little projects to work on along the way and it will really help your learning. They can be made up. They can be incredibly stupid or simple. Doesn't matter.. it's about figuring out how to use Python as a tool.


The Python to 'automate the boring tasks' book is great.. though it relies on many non-standard modules.. get to know the standard modules that come with Python as well. This is why folks are saying get Anaconda, btw, because it comes with a lot of extras people like, and Python already comes with more extras than many languages.

Others may not agree with me, but I would strongly suggest you avoid R as your first language. There are three reasons:
1) Wonky syntax that will not give you a clear understanding of the difference between values and expressions
2) Terrible tooling (RStudio is not a good IDE, and the others are worse)
3) Uninformative error/warning messages (tracking down bugs in R is a job for masochists)

(R has fantastic libraries for statistics and math, which is why people use it, but it's not a good language to learn on.)

Python is an excellent starting language. Go is also a good first language. Javascript can encourage some bad habits but is still a good place to start because you can do so much with it so quickly (I'd start with node.js first, so you can take advantage of new ES2015 syntax right away, and because you don't to figure out how web pages and html work. You can just start coding. And npm is awesome.)

I know I threw a lot of technical terminology at you, but if you'd like me to elaborate on any of these points, please let me know. My first language was C, by the way. Now that's a rough way to get into programming :)

As an R programmer for 16 years and counting I agree with most of this.  R started life as an open-source knockoff of S, and both were designed from their start as *special-purpose* languages for scientific and statistical computing. Python, on the other hand, was designed as a general-purpose scripting language, to fix some of the obvious ghastliness of Perl and to be easy for beginners *without* losing power. Totally different philosophies went into their creation.

That said:
1. RStudio does have everything other IDEs have - breakpoints, debugging, profiling, syntax highlighting, tool tips, code completion, folding, etc. The others aren't so much "worse" as they are unused - pretty much everyone who uses R abandoned them once RStudio got the features they depend on.
2. I don't know Python; when a Python code crashes and gives me a traceback I'm just as befuddled as I am by a hex dump of registers.
3. Over the past few years Python has acquired massive scientific and statistical computing libraries. It may never catch up with R in that respect but for *journalists* it's enough. Then again, so is Excel. ;-)

As someone who was programming several years before he decided to become a reporter, I’d like to make a couple of suggestions.

1. If you can, take time to play around. While I understand some people only want to learn to code to the extent that it helps them with their projects, I’ve found the best way to learn a programming language is to explore it. Every language boils down to some standard areas – Input/Output, Loops, Conditional branches, Data structures, Math functions, etc. When I’m learning a language, I’ll focus on each of these and try variations of examples in whatever book or books I’m working from, until I’m comfortable with them. Whenever I’ve tried to rush into doing something “productive,” I’ve had to rely on examples from other people rather than being able to come up with my own solutions.

 

2. Find short programming exercises. My favorite exercises are some of the standard computer games such as Hammurabi, Eliza, etc. They teach you a lot about parsing, program control, etc., yet they’re easy to understand and, typically, take an hour to a couple of hours to program. What I’ll do is take the original code, usually in Basic, and create a program outline and then convert that outline into code using the language I’m learning. Admittedly, Basic is one of the first languages I learned, so it’s easy for me to translate it, but it’s close enough to natural language that you can usually tell what each statement is trying to do (at least in these simple programs). Alternatively, some of the other recommendations include exercises more geared toward journalism.
